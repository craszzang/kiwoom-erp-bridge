[ GitHub 자동 배포 + 24시간 운영 안내 ]

■ 중요 (꼭 읽기)
  키움 OpenAPI는 Windows + 키움 설치 PC에서만 동작합니다.
  GitHub/클라우드에서 키움·매매를 24시간 돌리는 것은 불가능합니다 (무료 포함).

  가능한 것:
  - 코드 수정 → GitHub push → 고객 PC 자동 업데이트 (무료)
  - 고객 UI ↔ 호스트 브릿지 연결 (Tailscale, 무료)
  - 호스트 PC 24시간 켜두기 (미니PC/노트북 + 자동시작)

■ 역할
  | 역할 | 무엇이 24시간 필요? |
  | 호스트(당신 PC) | 키움 + 브릿지 (반드시 Windows PC) |
  | GitHub | 코드 저장 + ZIP 빌드 (무료, 키움 없음) |
  | 고객 PC | ERP UI만 (Tailscale로 호스트 연결) |

■ 최초 1회 (호스트)
  1. Git for Windows 설치
  2. github.com → New public repository
  3. 깃허브-최초설정.bat 실행 → 저장소 이름 입력
  4. GitHub 웹 → Actions → 워크플로우 성공 확인
  5. 호스트-자동시작등록.bat (로그인 시 브릿지 자동 실행)
  6. Windows 전원: 절전 안 함 / 디스플레이만 끄기 OK

■ 이후 수정 (Cursor에서)
  1. 코드 수정
  2. 터미널: git add . && git commit -m "fix" && git push
  3. GitHub Actions가 client.zip 릴리스 자동 생성
  4. 고객 PC: 재고실적_집계-원격.bat 실행 시 GitHub에서 자동 업데이트

■ 고객 config.yaml 예시
  bridge:
    host: '100.83.138.124'   # 호스트 Tailscale IP
    auto_sync: false
  update:
    source: github
    github_repo: YOUR_USER/YOUR_REPO
    github_branch: main

■ 24시간 안정 팁
  - Tailscale 호스트·고객 둘 다 Connected
  - 호스트: 브릿지 창 유지, 키움 로그인 유지
  - UPS(무정전) 있으면 더 좋음
  - 완전 무료 24h 클라우드 키움: 없음 (Windows VPS는 유료)
